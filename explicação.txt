EXPLICAÇÃO DO CÓDIGO DO PROJETO MINEIREZ

OBJETIVO GERAL

Este projeto implementa a parte lexical do compilador.
O analisador léxico, ou lexer, lê o código-fonte como texto e transforma esse texto em tokens.

Token é uma unidade de significado do código.
Exemplos:
- palavra-chave: uai, se, entao
- identificador: trem, valor, x
- número: 10, 200
- string: "trem bão"
- operador: =, >, <, +, -
- erro léxico: símbolo inválido, string não fechada, comentário não fechado

------------------------------------------------------------
1. ARQUIVO lexer.c
------------------------------------------------------------

Este arquivo é o coração do lexer.
Ele percorre o texto caractere por caractere e decide qual token deve ser retornado.

1.1 Includes

#include <stdio.h>
#include <ctype.h>
#include <string.h>

- stdio.h: usado em funções de entrada e saída padrão.
- ctype.h: fornece funções para testar caracteres, como isspace, isalpha e isdigit.
- string.h: usado para copiar e comparar strings, como strncpy e strcmp.

#include "token.h"
#include "lexer.h"

- token.h: contém a definição do tipo Token e do enum TipoToken.
- lexer.h: declara as funções iniciar_lexer e proximo_token.

1.2 Variáveis globais

static const char* codigo;
static int pos;

- codigo aponta para o texto fonte que será analisado.
- pos guarda a posição atual dentro da string.
- static limita o uso dessas variáveis apenas dentro de lexer.c.

A ideia é simples: o lexer sempre olha o caractere em codigo[pos] e avança conforme reconhece tokens.

1.3 Função copiar_lexema

static void copiar_lexema(Token* token, const char* inicio, int tamanho)

Essa função copia um trecho do código para dentro do token.
Ela recebe:
- token: onde o texto será guardado
- inicio: posição inicial do trecho
- tamanho: quantos caracteres devem ser copiados

O que ela faz:
- limita o tamanho máximo para evitar estourar o vetor token->valor
- percorre os caracteres do trecho
- troca quebras de linha e tab por espaço
- finaliza a string com '\0'

Essa função é útil para guardar o texto do token de forma segura.

1.4 Função copiar_comentario

static void copiar_comentario(Token* token, const char* inicio, int tamanho)

Essa função é parecida com copiar_lexema, mas foi criada para comentários.
Ela remove espaços em branco das bordas antes de copiar o texto.

Por que isso existe?
- Para o comentário aparecer mais limpo na saída
- Para evitar que o token venha com espaços extras no começo ou no fim

Exemplo:
- entrada: // comentario
- saída no token: comentario

1.5 Função criar_token

static Token criar_token(TipoToken tipo, const char* valor)

Essa função monta um token pronto.
Ela:
- define o tipo do token
- copia o texto recebido para token.valor
- garante o '\0' no final

Ela é usada principalmente para retornar TK_EOF e alguns TK_ERRO.

1.6 Função criar_erro

static Token criar_erro(const char* valor)

É uma função auxiliar que cria um token do tipo TK_ERRO.
Ela apenas chama criar_token com TK_ERRO.

Isso deixa o código mais organizado, porque vários erros diferentes podem ser retornados da mesma forma.

1.7 Função iniciar_lexer

void iniciar_lexer(const char* codigo_fonte)

Essa função inicia a análise léxica.
Ela recebe o texto completo do arquivo e prepara o lexer para começar do início.

O que ela faz:
- codigo recebe o texto fonte
- pos é zerado para começar na primeira posição

Sem essa função, o lexer não saberia qual texto analisar nem por onde começar.

1.8 Função proximo_token

Token proximo_token()

Essa é a função principal do lexer.
Cada vez que ela é chamada, ela retorna o próximo token encontrado no texto.

O fluxo geral é:
1. ignorar espaços em branco
2. verificar se o próximo trecho é comentário, identificador, número, string ou operador
3. se nada bater, retornar erro
4. se chegar no fim, retornar EOF

1.8.1 Ignorar espaços

if (isspace(atual))

- issapce verifica se o caractere é espaço, quebra de linha ou tab.
- Se for, o lexer avança pos e continua.

Isso é importante porque espaço não é token da linguagem, ele só separa os outros tokens.

1.8.2 Reconhecimento de comentários

O lexer reconhece três formas:
- # comentário de linha
- // comentário de linha
- /* comentário de bloco */

Comentário com #
- o lexer avança o caractere #
- lê até o fim da linha
- devolve TK_COMENTARIO

Comentário com //
- o lexer pula as duas barras
- lê até o fim da linha
- devolve TK_COMENTARIO

Comentário de bloco com /* ... */
- o lexer pula /*
- lê até encontrar */
- se encontrar, devolve TK_COMENTARIO
- se não encontrar, devolve TK_ERRO com comentario_nao_fechado

Por que isso é importante?
- Comentários devem ser reconhecidos para não serem tratados como erro
- O código mantém o texto comentado visível na saída, o que ajuda a debugar

1.8.3 Reconhecimento de identificadores e palavras-chave

if (isalpha(atual) || codigo[pos] == '_')

Esse trecho identifica nomes.
A regra é:
- começar com letra ou underscore
- continuar com letras, números ou underscore

Exemplos válidos:
- trem
- valor1
- minha_variavel

Depois de ler o texto, o lexer compara com as palavras reservadas:
- uai
- se
- entao
- fim
- fala

Se bater com uma palavra reservada, retorna o token correspondente.
Se não bater, retorna TK_ID.

1.8.4 Reconhecimento de números

if (isdigit(atual))

Esse trecho reconhece números inteiros.
Ele continua lendo enquanto houver dígitos.

Exemplo:
- entrada: 1234
- token: TK_NUM

Depois existe uma validação importante:
- se depois do número aparecer letra ou underscore, isso vira erro léxico

Exemplo de erro:
- 123abc
- 10_teste

Nesse caso o lexer retorna TK_ERRO com o texto completo do problema.

1.8.5 Reconhecimento de strings

if (codigo[pos] == '"')

Quando encontra aspas duplas, o lexer entende que começou uma string.
Ele:
- pula a aspas inicial
- lê até achar outra aspas
- não permite quebra de linha no meio da string

Se a string fechar corretamente:
- retorna TK_STRING

Se chegar ao fim da linha ou ao fim do arquivo sem fechar:
- retorna TK_ERRO com string_nao_fechada

1.8.6 Reconhecimento de operadores

O trecho:

switch (c)

verifica operadores simples da linguagem:
- =  -> TK_ATRIB
- >  -> TK_MAIOR
- <  -> TK_MENOR
- +  -> TK_SOMA
- -  -> TK_SUB

Se o caractere não for nenhum desses, ele cai no erro léxico.

1.8.7 Erro léxico genérico

Se nenhum caso anterior reconhecer o caractere, o lexer retorna TK_ERRO.
Isso cobre símbolos inválidos, por exemplo:
- @
- $
- &
- ?

Esse tipo de erro é importante porque ajuda a detectar caracteres que a linguagem não entende.

1.8.8 Fim do arquivo

Quando codigo[pos] encontra '\0', significa que o texto acabou.
Nesse caso o lexer retorna:
- TK_EOF

EOF quer dizer End Of File, ou fim do arquivo.

------------------------------------------------------------
5. NOVA SAÍDA EM LISTA
------------------------------------------------------------

A saída do programa agora não mostra mais apenas:

<TK_UAI, uai>

Agora ela imprime uma lista com mais informações sobre cada token:

- token: TK_UAI
	lexema: uai
	linha: 1
	coluna: 1

Campos da saída:

- token: nome do tipo do token
- lexema: texto guardado no token
- linha: linha onde o token começou
- coluna: coluna onde o token começou
- erro: mensagem exibida somente quando o token for TK_ERRO

Exemplo de erro:

- token: TK_ERRO
	lexema: 123
	linha: 3
	coluna: 1
	erro: numero_seguido_de_letra_ou_underscore

Isso ajuda a localizar exatamente onde o problema aconteceu no código-fonte.

------------------------------------------------------------
2. ARQUIVO main.c
------------------------------------------------------------

Esse arquivo cuida da execução do programa.
Ele lê o arquivo de entrada e chama o lexer repetidamente para imprimir os tokens.

2.1 Includes

#include <stdio.h>
#include <stdlib.h>

- stdio.h: usado para printf e fopen.
- stdlib.h: usado para malloc, free e exit.

#include "lexer.h"
#include "token.h"

- lexer.h: permite chamar iniciar_lexer e proximo_token
- token.h: dá acesso ao tipo Token e ao tipo_para_string

2.2 Função ler_arquivo

char* ler_arquivo(const char* nome_arquivo)

Essa função abre um arquivo e devolve seu conteúdo inteiro em uma string.

Passo a passo:
- fopen abre o arquivo em modo leitura
- se não conseguir abrir, mostra erro e encerra o programa
- fseek vai até o final do arquivo
- ftell pega o tamanho
- rewind volta para o começo
- malloc reserva memória suficiente para guardar o arquivo inteiro
- fread copia o conteúdo para a memória
- fecha o arquivo com fclose
- retorna o ponteiro para a string lida

Por que isso existe?
- Porque o lexer trabalha com texto em memória, não diretamente com arquivo.

2.3 Função main

int main(int argc, char *argv[])

Essa é a função de entrada do programa.
Ela recebe os argumentos da linha de comando.

Se o usuário não passar o nome do arquivo:
- mostra a mensagem de uso
- encerra com erro

Se o arquivo for informado:
- lê o conteúdo com ler_arquivo
- inicializa o lexer com iniciar_lexer
- chama proximo_token dentro de um laço
- imprime cada token encontrado

O laço faz isso:
- pega o próximo token
- imprime em formato de lista, mostrando token, lexema, linha, coluna e erro quando existir
- para quando encontrar TK_EOF

No final:
- libera a memória alocada com free

------------------------------------------------------------
3. ARQUIVO token.c
------------------------------------------------------------

Esse arquivo transforma o tipo do token em texto.
Ele é necessário para imprimir os tokens de forma legível.

3.1 Função tipo_para_string

const char* tipo_para_string(TipoToken tipo)

Essa função recebe um valor do enum TipoToken e devolve uma string correspondente.

Exemplos:
- TK_UAI -> "TK_UAI"
- TK_NUM -> "TK_NUM"
- TK_STRING -> "TK_STRING"
- TK_COMENTARIO -> "TK_COMENTARIO"
- TK_ERRO -> "TK_ERRO"

Se o tipo não existir na lista, retorna "DESCONHECIDO".

Por que isso é útil?
- Porque o printf do main precisa mostrar o nome do token, não só o número interno do enum.

------------------------------------------------------------
4. RESUMO DO FLUXO
------------------------------------------------------------

O programa funciona assim:
1. lê o arquivo .mz inteiro
2. inicializa o lexer
3. chama proximo_token várias vezes
4. imprime cada token encontrado
5. termina quando chega em TK_EOF

------------------------------------------------------------
6. COMO EXPLICAR NA AULA
------------------------------------------------------------

Se você quiser explicar de forma simples, pode dizer assim:
- O lexer lê o texto como se estivesse caminhando caractere por caractere.
- Quando encontra uma sequência conhecida, ele transforma em token.
- Quando encontra algo inválido, ele devolve erro.
- Comentários não fazem parte da lógica da linguagem, mas agora são reconhecidos e tratados separadamente.
- O arquivo main.c só faz a leitura do arquivo e mostra os tokens.
- O arquivo token.c converte o tipo num nome legível para aparecer na tela.
